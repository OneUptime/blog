# How to Use the Archive Provider to Create ZIP Files in Terraform

Author: [nawazdhandala](https://github.com/nawazdhandala)

Tags: Terraform, Archive Provider, ZIP, Lambda, Infrastructure as Code, Deployment

Description: Learn how to use the Terraform archive provider to create ZIP files for Lambda deployments, packaging source code, bundling configuration files, and creating deployment artifacts.

---

The archive provider in Terraform creates archive files (ZIP format) from source files or directories. Its most common use case is packaging Lambda function code for deployment, but it is also useful for bundling configuration files, creating deployment artifacts, and packaging any files that need to be uploaded to cloud services.

In this guide, we will explore the archive provider's data source archive_file, covering single file archives, directory archives, source content archives, and practical integration with AWS Lambda and other services.

## Provider Setup

```hcl
# main.tf
terraform {
  required_version = ">= 1.5.0"
  required_providers {
    archive = {
      source  = "hashicorp/archive"
      version = "~> 2.4"
    }
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
}

variable "environment" {
  type    = string
  default = "production"
}
```

## Creating a ZIP from a Single File

```hcl
# single-file.tf - Archive a single file
data "archive_file" "single" {
  type        = "zip"
  source_file = "${path.module}/src/handler.py"
  output_path = "${path.module}/dist/handler.zip"
}

output "archive_hash" {
  value = data.archive_file.single.output_base64sha256
}

output "archive_size" {
  value = data.archive_file.single.output_size
}
```

## Creating a ZIP from a Directory

```hcl
# directory.tf - Archive an entire directory
data "archive_file" "app" {
  type        = "zip"
  source_dir  = "${path.module}/src/app"
  output_path = "${path.module}/dist/app.zip"

  # Exclude patterns
  excludes = [
    "**/__pycache__/**",
    "**/*.pyc",
    "**/tests/**",
    "**/.git/**",
    "**/node_modules/**",
  ]
}
```

## Creating a ZIP from Inline Content

```hcl
# inline-content.tf - Archive content defined inline
data "archive_file" "inline" {
  type        = "zip"
  output_path = "${path.module}/dist/inline-lambda.zip"

  # Define file contents directly in Terraform
  source {
    content  = <<-PYTHON
      import json

      def handler(event, context):
          """Simple Lambda handler generated by Terraform."""
          return {
              'statusCode': 200,
              'body': json.dumps({
                  'message': 'Hello from ${var.environment}!',
                  'event': event
              })
          }
    PYTHON
    filename = "handler.py"
  }

  # Add additional files
  source {
    content = jsonencode({
      environment = var.environment
      version     = "1.0.0"
    })
    filename = "config.json"
  }
}
```

## Deploying a Lambda Function with Archive

```hcl
# lambda.tf - Complete Lambda deployment with archive
data "archive_file" "lambda" {
  type        = "zip"
  source_dir  = "${path.module}/lambda/src"
  output_path = "${path.module}/lambda/dist/function.zip"
}

resource "aws_lambda_function" "app" {
  function_name    = "app-function-${var.environment}"
  role             = aws_iam_role.lambda.arn
  handler          = "handler.lambda_handler"
  runtime          = "python3.11"
  timeout          = 30
  memory_size      = 256

  filename         = data.archive_file.lambda.output_path
  source_code_hash = data.archive_file.lambda.output_base64sha256

  environment {
    variables = {
      ENVIRONMENT = var.environment
    }
  }
}

resource "aws_iam_role" "lambda" {
  name = "lambda-role-${var.environment}"
  assume_role_policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Action = "sts:AssumeRole"
      Effect = "Allow"
      Principal = { Service = "lambda.amazonaws.com" }
    }]
  })
}

resource "aws_iam_role_policy_attachment" "lambda_basic" {
  role       = aws_iam_role.lambda.name
  policy_arn = "arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole"
}
```

## Archiving Multiple Lambda Functions

```hcl
# multi-lambda.tf - Package multiple Lambda functions
variable "lambda_functions" {
  type = map(object({
    source_dir = string
    handler    = string
    runtime    = string
    timeout    = number
    memory     = number
  }))
  default = {
    "api-handler" = {
      source_dir = "lambda/api"
      handler    = "handler.main"
      runtime    = "python3.11"
      timeout    = 30
      memory     = 256
    }
    "event-processor" = {
      source_dir = "lambda/processor"
      handler    = "index.handler"
      runtime    = "nodejs20.x"
      timeout    = 60
      memory     = 512
    }
    "data-transformer" = {
      source_dir = "lambda/transformer"
      handler    = "handler.transform"
      runtime    = "python3.11"
      timeout    = 120
      memory     = 1024
    }
  }
}

data "archive_file" "functions" {
  for_each = var.lambda_functions

  type        = "zip"
  source_dir  = "${path.module}/${each.value.source_dir}"
  output_path = "${path.module}/dist/${each.key}.zip"
}

resource "aws_lambda_function" "functions" {
  for_each = var.lambda_functions

  function_name    = "${each.key}-${var.environment}"
  role             = aws_iam_role.lambda.arn
  handler          = each.value.handler
  runtime          = each.value.runtime
  timeout          = each.value.timeout
  memory_size      = each.value.memory

  filename         = data.archive_file.functions[each.key].output_path
  source_code_hash = data.archive_file.functions[each.key].output_base64sha256
}
```

## Archiving with Dependencies

```hcl
# with-deps.tf - Package code with dependencies
# First, install dependencies (done outside Terraform)
# Then archive the result
data "archive_file" "with_deps" {
  type        = "zip"
  source_dir  = "${path.module}/lambda/build"
  output_path = "${path.module}/dist/with-deps.zip"

  # Only include the necessary files
  excludes = [
    "**/*.md",
    "**/tests/**",
    "**/__pycache__/**",
  ]
}
```

## Outputs

```hcl
# outputs.tf
output "archive_hashes" {
  description = "SHA256 hashes of all Lambda archives"
  value = {
    for k, v in data.archive_file.functions : k => v.output_base64sha256
  }
}

output "archive_sizes" {
  description = "Sizes of all Lambda archives in bytes"
  value = {
    for k, v in data.archive_file.functions : k => v.output_size
  }
}
```

## Conclusion

The archive provider is essential for packaging code and files for cloud deployments. By using the archive_file data source, you ensure that your deployment packages are always in sync with your source code and that Terraform can detect when the content has changed through the output_base64sha256 hash. For a deeper dive into Lambda-specific patterns, see our guide on [creating ZIP files for Lambda deployment](https://oneuptime.com/blog/post/2026-02-23-how-to-create-zip-files-for-lambda-deployment-with-terraform/view).
