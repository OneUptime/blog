# How to List All Istio Resources in a Cluster

Author: [nawazdhandala](https://github.com/nawazdhandala)

Tags: Istio, Kubernetes, Kubectl, Service Mesh, Cluster Management

Description: Practical methods for listing, querying, and inventorying all Istio custom resources across namespaces in your Kubernetes cluster.

---

Getting a full picture of what Istio resources exist in your cluster is something you'll need to do regularly. Whether you're auditing configurations, troubleshooting routing issues, or preparing for an upgrade, knowing how to efficiently list and filter Istio resources saves a lot of time.

## Listing All Istio CRD Types

First, find out what types of Istio resources your cluster supports:

```bash
kubectl api-resources | grep istio
```

This gives you a list of all registered Istio resource types:

```text
authorizationpolicies          ap           security.istio.io/v1                  true         AuthorizationPolicy
destinationrules               dr           networking.istio.io/v1                true         DestinationRule
envoyfilters                   ef           networking.istio.io/v1alpha3          true         EnvoyFilter
gateways                       gw           networking.istio.io/v1                true         Gateway
peerauthentications            pa           security.istio.io/v1                  true         PeerAuthentication
requestauthentications         ra           security.istio.io/v1                  true         RequestAuthentication
serviceentries                 se           networking.istio.io/v1                true         ServiceEntry
sidecars                       sc           networking.istio.io/v1                true         Sidecar
telemetries                    telemetry    telemetry.istio.io/v1                 true         Telemetry
virtualservices                vs           networking.istio.io/v1                true         VirtualService
wasmplugins                                 extensions.istio.io/v1alpha1          true         WasmPlugin
workloadentries                we           networking.istio.io/v1                true         WorkloadEntry
workloadgroups                 wg           networking.istio.io/v1                true         WorkloadGroup
```

Notice the short names column. These abbreviations make kubectl commands much faster to type.

## Listing All Instances Across All Namespaces

The simplest way to see all Istio resources is to query each type with the `-A` flag:

```bash
kubectl get vs,dr,gw,se,sc,ef,we,wg,pa,ap,ra,telemetry -A
```

This single command lists VirtualServices, DestinationRules, Gateways, ServiceEntries, Sidecars, EnvoyFilters, WorkloadEntries, WorkloadGroups, PeerAuthentications, AuthorizationPolicies, RequestAuthentications, and Telemetries across all namespaces.

The output looks like:

```text

default        virtualservice.networking.istio.io/bookinfo   ["bookinfo-gateway"]   ["*"]             5d
default        virtualservice.networking.istio.io/reviews                            ["reviews"]       5d

NAMESPACE      NAME                                              HOST      AGE
default        destinationrule.networking.istio.io/reviews       reviews   5d

NAMESPACE      NAME                                            AGE
default        gateway.networking.istio.io/bookinfo-gateway    5d
istio-system   gateway.networking.istio.io/istio-autogenerated 10d
```

## Getting Resource Counts

For a quick inventory, count resources by type:

```bash
echo "=== Istio Resource Inventory ==="
for resource in virtualservices destinationrules gateways serviceentries sidecars envoyfilters workloadentries workloadgroups peerauthentications authorizationpolicies requestauthentications telemetries wasmplugins; do
  count=$(kubectl get $resource -A --no-headers 2>/dev/null | wc -l | tr -d ' ')
  if [ "$count" -gt 0 ]; then
    printf "%-30s %s\n" "$resource" "$count"
  fi
done
```

Output:

```text
=== Istio Resource Inventory ===
virtualservices                12
destinationrules               8
gateways                       3
serviceentries                 5
sidecars                       2
peerauthentications            4
authorizationpolicies          7
telemetries                    1
```

## Filtering by Namespace

To see Istio resources in a specific namespace:

```bash
kubectl get vs,dr,gw,se,sc,pa,ap -n my-namespace
```

Or to get a per-namespace breakdown:

```bash
for ns in $(kubectl get namespaces -o jsonpath='{.items[*].metadata.name}'); do
  count=$(kubectl get vs,dr,gw,se,sc,pa,ap -n $ns --no-headers 2>/dev/null | wc -l | tr -d ' ')
  if [ "$count" -gt 0 ]; then
    echo "$ns: $count resources"
  fi
done
```

## Using Labels to Filter

If your resources are labeled (and they should be), you can filter by labels:

```bash
# Find all resources owned by a specific team
kubectl get vs,dr -A -l team=backend

# Find all resources managed by ArgoCD
kubectl get vs,dr,gw -A -l app.kubernetes.io/managed-by=argocd

# Find resources for a specific application
kubectl get vs,dr,se -A -l app=my-service
```

## Detailed Output Formats

For more information, use wider output:

```bash
kubectl get vs -A -o wide
```

For YAML output of a specific resource:

```bash
kubectl get vs my-route -n default -o yaml
```

For JSON output (useful for scripting):

```bash
kubectl get vs -A -o json | python3 -m json.tool
```

Custom columns let you extract exactly the fields you need:

```bash
kubectl get vs -A -o custom-columns=\
NAMESPACE:.metadata.namespace,\
NAME:.metadata.name,\
HOSTS:.spec.hosts,\
GATEWAYS:.spec.gateways
```

## Using istioctl for Resource Discovery

The `istioctl` tool provides higher-level views of Istio configuration:

```bash
# Show what configuration applies to a specific pod
istioctl x describe pod <pod-name> -n <namespace>
```

This shows all the Istio resources that affect a specific workload, including inherited policies from parent namespaces.

```bash
# Show proxy status for all sidecars
istioctl proxy-status
```

This lists all connected proxies and their sync status:

```text

reviews-v1-xxx.default                   Kubernetes  SYNCED  SYNCED  SYNCED  SYNCED  -       istiod-abc-xyz
```

## Finding Resources That Reference a Specific Service

To find all VirtualServices that route to a specific service:

```bash
kubectl get vs -A -o json | python3 -c "
import json, sys
data = json.load(sys.stdin)
for item in data.get('items', []):
    name = item['metadata']['name']
    ns = item['metadata']['namespace']
    spec = json.dumps(item.get('spec', {}))
    if 'my-service' in spec:
        print(f'{ns}/{name}')
"
```

For DestinationRules:

```bash
kubectl get dr -A -o jsonpath='{range .items[*]}{.metadata.namespace}/{.metadata.name}: {.spec.host}{"\n"}{end}' | grep my-service
```

## Exporting a Full Inventory

Create a comprehensive inventory report:

```bash
#!/bin/bash
echo "Istio Cluster Inventory Report"
echo "Generated: $(date)"
echo "Cluster: $(kubectl config current-context)"
echo "=============================="

echo ""
echo "## Istio Version"
istioctl version 2>/dev/null || echo "istioctl not available"

echo ""
echo "## Control Plane"
kubectl get pods -n istio-system

echo ""
echo "## CRD Versions"
kubectl get crds | grep istio | while read name rest; do
  versions=$(kubectl get crd $name -o jsonpath='{.spec.versions[*].name}')
  printf "%-60s %s\n" "$name" "$versions"
done

echo ""
echo "## Resource Summary"
for resource in virtualservices destinationrules gateways serviceentries sidecars envoyfilters peerauthentications authorizationpolicies requestauthentications telemetries wasmplugins workloadentries workloadgroups; do
  count=$(kubectl get $resource -A --no-headers 2>/dev/null | wc -l | tr -d ' ')
  printf "%-30s %s\n" "$resource" "$count"
done

echo ""
echo "## Sidecar Injection Status"
kubectl get namespaces -L istio-injection -L istio.io/rev

echo ""
echo "## Proxy Status"
istioctl proxy-status 2>/dev/null
```

Save this as `istio-inventory.sh` and run it whenever you need a snapshot of your mesh configuration.

## Watching for Changes

To monitor Istio resource changes in real time:

```bash
kubectl get vs,dr,gw,pa,ap -A --watch
```

This streams updates as resources are created, modified, or deleted. Useful during deployments or when troubleshooting configuration propagation.

For a more structured approach, you can watch events related to Istio resources:

```bash
kubectl get events -A --field-selector reason=Synced --watch
```

Knowing how to quickly inventory and query your Istio resources is a foundational skill for mesh operations. These commands and scripts give you visibility into what's configured, where it's configured, and how it all fits together.
