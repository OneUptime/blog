# How to Troubleshoot Installation Issues with Calico on Windows Nodes with Rancher

Author: [nawazdhandala](https://github.com/nawazdhandala)

Tags: Calico, Kubernetes, Windows, Rancher, Networking, CNI, Troubleshooting

Description: A diagnostic guide for resolving Calico and networking failures on Windows nodes in Rancher-managed Kubernetes clusters.

---

## Introduction

Troubleshooting Calico on Windows nodes in Rancher-managed clusters requires checking Rancher's node registration state in addition to the standard Calico and Windows networking diagnostics. A Windows node that is registered with Rancher but has networking problems may show as Active in Rancher while being unable to run pods correctly. This guide covers Rancher-specific checks alongside standard Windows Calico diagnostics.

Rancher's cluster agent (cattle-cluster-agent) must be able to communicate with Windows nodes for management purposes, and this communication uses the same pod network that Calico manages. If Calico is misconfigured, Rancher may lose management visibility into Windows nodes, making remote troubleshooting harder.

## Prerequisites

- Rancher-managed cluster with Windows nodes
- Access to Rancher UI and kubectl
- PowerShell access to Windows nodes

## Step 1: Check Rancher Node Status

In Rancher UI:
- Navigate to **Cluster** > **Nodes**
- Check Windows node status and look for error messages

Via CLI:

```bash
kubectl describe node <windows-node> | grep -A10 "Conditions:"
```

## Step 2: Check Rancher Agent Pod on Windows

```bash
kubectl get pods -n cattle-system -o wide | grep <windows-node>
kubectl logs -n cattle-system <rancher-agent-pod-on-windows-node> --tail=30
```

If the Rancher agent pod cannot start, Calico may not be initialized on the Windows node.

## Step 3: Check Calico Windows Components

```bash
kubectl get pods -n calico-system -o wide | grep <windows-node>
kubectl logs -n calico-system <calico-node-windows-pod> --tail=50
```

## Step 4: Verify Windows Node Can Reach Rancher Server

```powershell
# Test connectivity to Rancher server
Test-NetConnection -ComputerName rancher.example.com -Port 443
Test-NetConnection -ComputerName rancher.example.com -Port 8443
```

If Rancher is unreachable, check that Calico is not blocking egress from the Windows node to the Rancher server's IP range.

## Step 5: Diagnose HNS Issues

```powershell
# Check HNS state
Get-HnsNetwork
Get-HnsEndpoint | Measure-Object

# If HNS is in a bad state
Stop-Service HNS -Force
Start-Service HNS
```


## Conclusion

Troubleshooting Calico on Windows nodes in Rancher clusters adds Rancher agent communication checks to the standard Calico and Windows HNS diagnostics. The Rancher agent's ability to communicate with Windows nodes depends on Calico's correct operation, creating a dependency that must be resolved before Rancher management features become available. In extreme cases, re-registering the Windows node with Rancher is the fastest path to recovery.
