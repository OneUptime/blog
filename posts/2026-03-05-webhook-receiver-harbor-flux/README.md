# How to Configure Webhook Receiver for Harbor in Flux

Author: [nawazdhandala](https://github.com/nawazdhandala)

Tags: Flux CD, GitOps, Kubernetes, Webhooks, Harbor, Receiver, Container Registry

Description: Learn how to configure a Flux Receiver to accept Harbor webhooks and trigger reconciliation when new container images are pushed.

---

Harbor is a popular open-source container registry that supports webhook notifications for various events, including image pushes. By configuring a Flux Receiver with type `harbor`, you can trigger immediate reconciliation of image-related resources when new images are pushed to your Harbor registry. This guide walks through the complete setup.

## Prerequisites

- A Kubernetes cluster with Flux CD installed, including the notification controller
- A Harbor registry instance with at least one project
- Admin access to the Harbor project for webhook configuration
- An ingress controller or load balancer to expose the receiver endpoint

## Step 1: Create the Webhook Secret

Create a Kubernetes secret for authenticating Harbor webhook requests.

```bash
# Generate a random token
TOKEN=$(head -c 12 /dev/urandom | shasum | cut -d ' ' -f1)

# Create the secret in flux-system namespace
kubectl create secret generic harbor-webhook-secret \
  --namespace=flux-system \
  --from-literal=token=$TOKEN

# Save the token for Harbor configuration
echo "Webhook token: $TOKEN"
```

## Step 2: Create the Receiver Resource

Define a Receiver with type `harbor`.

```yaml
# Receiver for Harbor webhook events
apiVersion: notification.toolkit.fluxcd.io/v1
kind: Receiver
metadata:
  name: harbor-receiver
  namespace: flux-system
spec:
  # Harbor webhook type
  type: harbor
  # Harbor push artifact events
  events:
    - "PUSH_ARTIFACT"
  # Secret for webhook authentication
  secretRef:
    name: harbor-webhook-secret
  # Resources to reconcile on image push
  resources:
    - kind: ImageRepository
      name: my-app
```

Apply the receiver.

```bash
# Apply the receiver configuration
kubectl apply -f harbor-receiver.yaml

# Check the receiver status
kubectl get receivers -n flux-system
```

## Step 3: Get the Webhook URL

Retrieve the webhook path generated by Flux.

```bash
# Get the webhook URL path
kubectl get receiver harbor-receiver -n flux-system -o jsonpath='{.status.webhookPath}'
```

## Step 4: Expose the Receiver Endpoint

Create an ingress to make the endpoint reachable from Harbor.

```yaml
# Ingress for Harbor webhook receiver
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: harbor-webhook-ingress
  namespace: flux-system
  annotations:
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
spec:
  rules:
    - host: flux-webhook.example.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: notification-controller
                port:
                  number: 80
  tls:
    - hosts:
        - flux-webhook.example.com
      secretName: webhook-tls
```

## Step 5: Configure the Harbor Webhook

Set up the webhook in Harbor.

1. Log in to Harbor and navigate to your project
2. Go to **Webhooks** in the left sidebar
3. Click **+ New Webhook Policy**
4. Set **Name** to "Flux CD Receiver"
5. Set **Notify Type** to HTTP
6. Set **Endpoint URL** to `https://flux-webhook.example.com/<webhook-path>`
7. Set **Auth Header** to the token value from Step 1
8. Under **Event Type**, select **Artifact pushed**
9. Click **Save**

## Step 6: Handle Multiple Image Repositories

Configure the receiver to trigger scans for multiple image repositories.

```yaml
# Receiver triggering multiple image repositories from Harbor
apiVersion: notification.toolkit.fluxcd.io/v1
kind: Receiver
metadata:
  name: harbor-multi-receiver
  namespace: flux-system
spec:
  type: harbor
  events:
    - "PUSH_ARTIFACT"
  secretRef:
    name: harbor-webhook-secret
  resources:
    # Frontend application image
    - kind: ImageRepository
      name: frontend
    # Backend API image
    - kind: ImageRepository
      name: backend-api
    # Worker service image
    - kind: ImageRepository
      name: worker
```

## Step 7: Verify and Test

Confirm the receiver is working by pushing an image to Harbor.

```bash
# Check receiver readiness
kubectl get receiver harbor-receiver -n flux-system

# Describe the receiver for details
kubectl describe receiver harbor-receiver -n flux-system

# Watch notification controller logs
kubectl logs -n flux-system deploy/notification-controller -f

# Push a test image to Harbor
docker tag my-app:latest harbor.example.com/myproject/my-app:test
docker push harbor.example.com/myproject/my-app:test

# Verify the ImageRepository was scanned
kubectl get imagerepository my-app -n flux-system
```

## Troubleshooting

If Harbor webhooks are not working, check these areas.

```bash
# Verify the receiver status
kubectl get receiver harbor-receiver -n flux-system -o yaml

# Check notification controller logs
kubectl logs -n flux-system deploy/notification-controller | grep -i "harbor\|receiver"

# Verify the secret
kubectl get secret harbor-webhook-secret -n flux-system -o jsonpath='{.data.token}' | base64 -d

# Test endpoint connectivity from within the cluster
kubectl run curl-test --image=curlimages/curl --rm -it -- curl -I https://flux-webhook.example.com/
```

In Harbor, check the webhook policy's **Recent Activity** section to view delivery attempts and their results. This shows the HTTP status codes returned by the Flux receiver endpoint.

## Summary

Configuring a Harbor webhook receiver in Flux connects your private container registry directly to your GitOps pipeline. When new images are pushed to Harbor, the receiver triggers immediate scanning of ImageRepository resources, accelerating the image update workflow. The setup involves creating a shared secret, defining a Receiver resource with type `harbor`, exposing the endpoint, and configuring the webhook in Harbor's project settings.
