# How to Upgrade PostgreSQL Versions on RHEL Using pg_upgrade

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: RHEL, PostgreSQL, Pg_upgrade, Database, Version Upgrade, Migration

Description: Upgrade PostgreSQL to a new major version on RHEL using pg_upgrade for an in-place upgrade that is faster than a dump-and-restore approach.

---

PostgreSQL major version upgrades (for example, 15 to 16) require either a full dump-and-restore or using the pg_upgrade tool. pg_upgrade performs an in-place upgrade that is significantly faster because it does not need to dump and reload all data.

## Install the New PostgreSQL Version

```bash
# Install the PostgreSQL repository for the new version
sudo dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-9-x86_64/pgdg-redhat-repo-latest.noarch.rpm

# Disable the built-in PostgreSQL module
sudo dnf module disable -y postgresql

# Install the new version (e.g., PostgreSQL 16)
sudo dnf install -y postgresql16-server postgresql16
```

## Prepare for the Upgrade

```bash
# Check current version
sudo -u postgres /usr/bin/psql -c "SELECT version();"

# Initialize the new version's data directory
sudo /usr/pgsql-16/bin/postgresql-16-setup initdb

# Verify both data directories exist
ls -la /var/lib/pgsql/data/       # Old version (e.g., 15)
ls -la /var/lib/pgsql/16/data/    # New version (16)
```

## Stop the Old PostgreSQL Instance

```bash
# Stop the old PostgreSQL service
sudo systemctl stop postgresql

# Verify it is stopped
systemctl status postgresql
```

## Run the Upgrade Check

```bash
# Run pg_upgrade in check mode first (does not make changes)
sudo -u postgres /usr/pgsql-16/bin/pg_upgrade \
    --old-datadir=/var/lib/pgsql/data \
    --new-datadir=/var/lib/pgsql/16/data \
    --old-bindir=/usr/bin \
    --new-bindir=/usr/pgsql-16/bin \
    --check

# Review the output for any errors or required actions
```

## Run the Upgrade

```bash
# Perform the actual upgrade
sudo -u postgres /usr/pgsql-16/bin/pg_upgrade \
    --old-datadir=/var/lib/pgsql/data \
    --new-datadir=/var/lib/pgsql/16/data \
    --old-bindir=/usr/bin \
    --new-bindir=/usr/pgsql-16/bin

# For large databases, use link mode (faster, but old cluster becomes unusable)
# sudo -u postgres /usr/pgsql-16/bin/pg_upgrade \
#     --old-datadir=/var/lib/pgsql/data \
#     --new-datadir=/var/lib/pgsql/16/data \
#     --old-bindir=/usr/bin \
#     --new-bindir=/usr/pgsql-16/bin \
#     --link
```

## Copy Configuration Files

```bash
# Copy your customized configuration from the old cluster
sudo cp /var/lib/pgsql/data/postgresql.conf /var/lib/pgsql/16/data/postgresql.conf
sudo cp /var/lib/pgsql/data/pg_hba.conf /var/lib/pgsql/16/data/pg_hba.conf
sudo chown postgres:postgres /var/lib/pgsql/16/data/postgresql.conf
sudo chown postgres:postgres /var/lib/pgsql/16/data/pg_hba.conf

# Review and update any deprecated settings for the new version
# Check the PostgreSQL release notes for removed/changed parameters
```

## Start the New Version

```bash
# Start the new PostgreSQL 16 service
sudo systemctl start postgresql-16
sudo systemctl enable postgresql-16

# Verify it is running with the new version
sudo -u postgres /usr/pgsql-16/bin/psql -c "SELECT version();"
```

## Post-Upgrade Tasks

```bash
# Run the optimizer statistics update (generated by pg_upgrade)
sudo -u postgres /usr/pgsql-16/bin/vacuumdb --all --analyze-in-stages

# If pg_upgrade created a delete script for old cluster files:
# Review it first, then run it
cat /var/lib/pgsql/delete_old_cluster.sh
# sudo -u postgres bash /var/lib/pgsql/delete_old_cluster.sh
```

## Remove the Old Version

```bash
# After verifying everything works, remove the old PostgreSQL packages
sudo dnf remove -y postgresql-server postgresql

# Clean up old data directory (only after confirming the upgrade is successful)
# sudo rm -rf /var/lib/pgsql/data
```

## Rollback Plan

If the upgrade fails:

```bash
# Stop the new version
sudo systemctl stop postgresql-16

# Start the old version (only if you did NOT use --link mode)
sudo systemctl start postgresql
```

Always take a full backup before starting the upgrade process. pg_upgrade is reliable, but having a backup provides a safety net if something unexpected happens.
