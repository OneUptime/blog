# How to Set Up Node-RED on Ubuntu for IoT Workflows

Author: [nawazdhandala](https://github.com/nawazdhandala)

Tags: Ubuntu, Node-RED, IoT, Automation, MQTT

Description: A complete guide to installing and configuring Node-RED on Ubuntu, securing the dashboard, and building IoT workflows that process MQTT messages and automate device responses.

---

Node-RED is a flow-based programming tool that lets you wire together devices, APIs, and services using a browser-based visual editor. It runs on Node.js and is particularly well-suited for IoT workflows: receive data from sensors over MQTT, process it, store it in a database, trigger alerts, and control actuators - all without writing traditional code. That said, JavaScript function nodes let you add custom logic when the built-in nodes are not enough.

## Prerequisites

- Ubuntu 20.04 or 22.04
- sudo privileges
- Node.js 18 or 20 (we will install it via nvm or NodeSource)

## Installing Node.js

Node-RED requires Node.js. The version from Ubuntu's default repositories is often outdated. Use the NodeSource repository:

```bash
# Install Node.js 20.x via NodeSource
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Verify
node --version  # Should be v20.x.x
npm --version
```

Alternatively, use nvm for more flexibility:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20
```

## Installing Node-RED

```bash
# Install Node-RED globally
sudo npm install -g --unsafe-perm node-red

# Verify installation
node-red --version
```

## Running Node-RED

For a quick test, run Node-RED directly:

```bash
node-red
# Visit http://localhost:1880 in your browser
```

For a production setup, create a systemd service:

```bash
# Create a dedicated user
sudo useradd --system --home /var/lib/node-red --shell /bin/false nodered
sudo mkdir -p /var/lib/node-red
sudo chown nodered:nodered /var/lib/node-red

# Create the service
sudo tee /etc/systemd/system/node-red.service << 'EOF'
[Unit]
Description=Node-RED
Documentation=https://nodered.org
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=nodered
Group=nodered

# Node-RED userdir
Environment="HOME=/var/lib/node-red"
WorkingDirectory=/var/lib/node-red

# Start Node-RED
ExecStart=/usr/local/bin/node-red --userDir /var/lib/node-red

Restart=on-failure
RestartSec=5

# Resource limits
MemoryMax=512M

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable node-red
sudo systemctl start node-red

# Check logs
sudo journalctl -u node-red -f
```

## Securing Node-RED

The default Node-RED installation has no authentication. Fix this before exposing it to a network.

```bash
# Generate a bcrypt password hash
node-red-admin hash-pw
# Enter your password when prompted
# Output: $2b$08$...  (save this hash)
```

Edit the Node-RED settings file:

```bash
sudo nano /var/lib/node-red/settings.js
```

Find the `adminAuth` section and configure it:

```javascript
// /var/lib/node-red/settings.js

module.exports = {
    // Bind to all interfaces or just localhost
    uiHost: "0.0.0.0",
    uiPort: 1880,

    // Authentication for the editor
    adminAuth: {
        type: "credentials",
        users: [{
            username: "admin",
            // Paste the hash generated by node-red-admin hash-pw
            password: "$2b$08$your-hashed-password-here",
            permissions: "*"
        }, {
            username: "viewer",
            password: "$2b$08$viewer-password-hash",
            permissions: "read"  // Read-only access
        }]
    },

    // Optional: secure the Node-RED dashboard UI separately
    // (dashboard is at /ui by default)
    httpNodeAuth: {
        user: "user",
        pass: "$2b$08$dashboard-password-hash"
    },

    // HTTPS (recommended in production)
    // https: {
    //     key: fs.readFileSync('/etc/letsencrypt/live/yourdomain.com/privkey.pem'),
    //     cert: fs.readFileSync('/etc/letsencrypt/live/yourdomain.com/cert.pem')
    // },

    // Logging
    logging: {
        console: {
            level: "info",
            metrics: false,
            audit: false
        }
    },

    // Flow file settings
    flowFile: 'flows.json',
    credentialSecret: "your-secret-key-for-credential-encryption",

    // Node modules
    functionGlobalContext: {
        // Variables available in all function nodes
        // os: require('os'),
    },

    // Timeout for HTTP requests made from HTTP In nodes
    httpRequestTimeout: 120000,

    // Editor settings
    editorTheme: {
        projects: {
            enabled: true  // Enable version control via Git
        }
    }
};
```

```bash
sudo systemctl restart node-red
```

## Installing Additional Nodes

Node-RED has a large package ecosystem. Install nodes via the Palette Manager in the UI, or from the command line:

```bash
# Navigate to the userdir
cd /var/lib/node-red

# Install popular IoT nodes
sudo -u nodered npm install \
  node-red-contrib-mqtt-broker \   # Embedded MQTT broker
  node-red-dashboard \             # Web dashboard
  node-red-node-sqlite \           # SQLite database
  node-red-contrib-influxdb \      # InfluxDB time-series
  node-red-contrib-http-request \  # Enhanced HTTP node
  node-red-contrib-home-assistant  # Home Assistant integration

sudo systemctl restart node-red
```

## Building an IoT Workflow: MQTT Sensor to Dashboard

Here is a practical example: receiving temperature readings from a sensor over MQTT, processing them, and displaying on a dashboard.

### Example Flow (JSON to import via Node-RED UI)

```json
[
    {
        "id": "mqtt-in-node",
        "type": "mqtt in",
        "name": "Temperature Sensor",
        "topic": "sensors/+/temperature",
        "qos": "1",
        "broker": "mqtt-broker-config",
        "x": 100,
        "y": 100,
        "wires": [["json-parse"]]
    },
    {
        "id": "json-parse",
        "type": "json",
        "name": "Parse JSON",
        "x": 300,
        "y": 100,
        "wires": [["temperature-check"]]
    },
    {
        "id": "temperature-check",
        "type": "function",
        "name": "Check Threshold",
        "func": "// msg.payload = { sensor_id: '..', temperature: 25.4, humidity: 60 }\nvar temp = msg.payload.temperature;\nvar sensorId = msg.payload.sensor_id;\n\n// Tag the message with metadata\nmsg.topic = sensorId;\nmsg.timestamp = new Date().toISOString();\n\n// Alert if temperature is too high\nif (temp > 35) {\n    msg.alert = true;\n    node.warn('High temperature: ' + temp + ' from ' + sensorId);\n}\n\n// Prepare for InfluxDB\nmsg.influx = [\n    { measurement: 'temperature',\n      fields: { value: temp },\n      tags: { sensor: sensorId } }\n];\n\nreturn msg;",
        "x": 500,
        "y": 100,
        "wires": [["dashboard-gauge", "influx-write", "alert-switch"]]
    }
]
```

### Configuring the MQTT Connection

In Node-RED:
1. Double-click an MQTT In node
2. Click the pencil icon next to "Server"
3. Configure your MQTT broker (Mosquitto):
   - Server: `localhost` (or your broker IP)
   - Port: `1883`
   - Client ID: `node-red-client`
   - Add username/password if your broker requires auth

```bash
# Install Mosquitto as the MQTT broker
sudo apt-get install -y mosquitto mosquitto-clients

# Basic Mosquitto config
sudo tee /etc/mosquitto/conf.d/local.conf << 'EOF'
listener 1883 localhost
allow_anonymous true

# For external access with auth:
# listener 1883 0.0.0.0
# allow_anonymous false
# password_file /etc/mosquitto/passwd
EOF

sudo systemctl restart mosquitto

# Test: publish a sensor reading
mosquitto_pub -h localhost -t "sensors/sensor-01/temperature" \
  -m '{"sensor_id":"sensor-01","temperature":23.5,"humidity":55}'
```

## Building a Dashboard

With `node-red-dashboard` installed:

1. Drag a **dashboard** > **gauge** node onto the canvas
2. Connect it to the function node output
3. Configure: Group: "Sensors", Label: "Temperature", Units: "°C", Range: 0-50
4. Deploy (red Deploy button in top right)
5. Access dashboard at `http://your-server:1880/ui`

## Exporting and Importing Flows

Back up your flows:

```bash
# Flows are stored as JSON
sudo cat /var/lib/node-red/flows.json

# Back up regularly
sudo cp /var/lib/node-red/flows.json /var/backups/node-red-flows-$(date +%Y%m%d).json
```

Import a flow:
1. Click the hamburger menu in Node-RED
2. Import > Select file or paste JSON
3. Deploy

## Using Environment Variables

Keep credentials out of your flows by using environment variables:

```bash
# Add to the systemd service
sudo systemctl edit node-red
```

```ini
[Service]
Environment="MQTT_HOST=mqtt.internal"
Environment="MQTT_USER=nodered"
Environment="MQTT_PASS=secret"
Environment="INFLUX_TOKEN=your-influx-token"
```

Access in flow nodes: use `${MQTT_HOST}` in MQTT broker config, or in function nodes:

```javascript
// Access env vars in function nodes
var mqttHost = env.get("MQTT_HOST");
```

## Troubleshooting

**Node-RED not starting:**
```bash
# Check for port conflicts
sudo ss -tlnp | grep 1880

# Check permission on userdir
ls -la /var/lib/node-red/
sudo chown -R nodered:nodered /var/lib/node-red/
```

**MQTT connection refused:**
```bash
# Check Mosquitto is running
systemctl status mosquitto

# Test connection manually
mosquitto_sub -h localhost -t "test/#" -v
```

**Nodes missing after package install:**
```bash
# Ensure packages installed in the correct userdir
cd /var/lib/node-red
sudo -u nodered npm list --depth=0
sudo systemctl restart node-red
```
