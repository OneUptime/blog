# How to Install BookStack Documentation Platform on Ubuntu

Author: [nawazdhandala](https://github.com/nawazdhandala)

Tags: Ubuntu, BookStack, Self-Hosted, Documentation, Knowledge Management

Description: Install BookStack on Ubuntu with MySQL and Apache or Nginx, configure it for team documentation with books, chapters, and pages, and set up LDAP or social authentication.

---

BookStack is an open-source, self-hosted documentation platform built on Laravel. It organizes content hierarchically: Shelves contain Books, Books contain Chapters, and Chapters contain Pages. This structure works well for technical documentation, runbooks, internal wikis, and knowledge bases. The editor is WYSIWYG by default with Markdown support available, making it accessible to both technical and non-technical writers.

## Prerequisites

- Ubuntu 22.04 or 24.04
- PHP 8.1 or higher with required extensions
- MySQL 5.7+ or MariaDB 10.6+
- A web server (Apache or Nginx)
- Composer (PHP dependency manager)

## Installing Required Software

```bash
# Update system
sudo apt update && sudo apt upgrade -y

# Install PHP and required extensions
sudo apt install -y \

sudo apt install -y nginx

# Install MariaDB
sudo apt install -y mariadb-server

curl -sS https://getcomposer.org/installer | sudo php -- --install-dir=/usr/local/bin --filename=composer

# Verify installations
php --version
composer --version
mysql --version
```

## Setting Up the Database

```bash
# Secure MariaDB installation
sudo mysql_secure_installation

# Create BookStack database and user
sudo mysql -u root -p << 'EOF'
CREATE DATABASE bookstack;
CREATE USER 'bookstack'@'localhost' IDENTIFIED BY 'strong-db-password';
GRANT ALL PRIVILEGES ON bookstack.* TO 'bookstack'@'localhost';
FLUSH PRIVILEGES;
EXIT;
EOF
```

## Installing BookStack

```bash
# Install Git if not present
sudo apt install -y git

# Clone BookStack to the web directory
cd /var/www
sudo git clone https://github.com/BookStackApp/BookStack.git --branch release --single-branch bookstack

# Navigate to BookStack directory
cd /var/www/bookstack

# Install PHP dependencies
sudo composer install --no-dev

# Copy the example environment file
sudo cp .env.example .env

# Generate the application key
sudo php artisan key:generate

# Set proper ownership
sudo chown -R www-data:www-data /var/www/bookstack
sudo chmod -R 755 /var/www/bookstack
sudo chmod -R 775 /var/www/bookstack/storage /var/www/bookstack/bootstrap/cache
```

## Configuring BookStack

```bash
sudo nano /var/www/bookstack/.env
```

```env
# BookStack environment configuration

APP_ENV=production
APP_DEBUG=false
APP_KEY=  # Generated by artisan key:generate - do not change

# Application URL - must match your domain exactly
APP_URL=https://docs.example.com

# Database connection
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=bookstack
DB_USERNAME=bookstack
DB_PASSWORD=strong-db-password

# Cache and session (file works for single-server installs)
CACHE_DRIVER=file
SESSION_DRIVER=database

# File uploads location
STORAGE_TYPE=local

# Email configuration
MAIL_DRIVER=smtp
MAIL_HOST=smtp.example.com
MAIL_PORT=587
MAIL_USERNAME=bookstack@example.com
MAIL_PASSWORD=smtp-password
MAIL_ENCRYPTION=tls
MAIL_FROM=bookstack@example.com
MAIL_FROM_NAME="BookStack Docs"

# Default language
APP_LANG=en

# Authentication type: standard (default) or ldap
AUTH_METHOD=standard

# Disable user registration (invite-only)
# REGISTRATION_ENABLED=false
```

After editing `.env`, run the database migrations:

```bash
# Run database migrations and seed initial data
cd /var/www/bookstack
sudo -u www-data php artisan migrate --force
```

## Configuring Nginx

```bash
sudo nano /etc/nginx/sites-available/bookstack
```

```nginx
# BookStack Nginx configuration
server {
    listen 80;
    server_name docs.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name docs.example.com;

    root /var/www/bookstack/public;
    index index.php;

    ssl_certificate     /etc/letsencrypt/live/docs.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/docs.example.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;

    # Allow file uploads (attachments and images)
    client_max_body_size 100m;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        fastcgi_read_timeout 300;
    }

    # Block access to sensitive directories
    location ~ ^/(storage|vendor|tests|\.git) {
        deny all;
        return 404;
    }

    # Static asset caching
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|woff2?)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/bookstack /etc/nginx/sites-enabled/

# Get SSL certificate
sudo apt install -y certbot python3-certbot-nginx
sudo certbot certonly --nginx -d docs.example.com

sudo nginx -t && sudo systemctl reload nginx
sudo systemctl restart php8.2-fpm
```

## Initial Login

Visit `https://docs.example.com` and log in with:
- Email: `admin@admin.com`
- Password: `password`

**Change both the email and password immediately** in your profile settings.

## Setting Up the Application Structure

BookStack's hierarchy:
- **Shelves** - top level, group related books (e.g., "Engineering", "HR Policies")
- **Books** - a collection of chapters and pages (e.g., "Infrastructure Runbooks")
- **Chapters** - optional grouping of pages within a book
- **Pages** - individual documentation pages

### Creating Your First Book

1. Click "New Book" from the dashboard or a shelf
2. Name: `Infrastructure Runbooks`
3. Add a description
4. Click Create

Then add chapters and pages within the book.

## User Management and Permissions

BookStack has a role-based permission system. Default roles:
- **Admin** - full access to all content and settings
- **Editor** - can create and edit content
- **Viewer** - read-only access

Create custom roles in Settings > Roles:

```text
1. Settings > Roles > New Role
2. Name: "Engineering Team"
3. Permissions:
   - View All Books: enabled
   - Create Pages: enabled
   - Edit Own Pages: enabled
   - Delete Own Pages: enabled
4. Save
```

Assign roles to users in Settings > Users > (select user) > Roles.

### Per-Content Permissions

For granular control, each Book, Chapter, and Page has its own permission settings:
1. Click the three-dot menu on any book
2. Select "Permissions"
3. Enable "Custom Permissions"
4. Set read/write/delete access per role

This lets you have public documentation and private internal docs in the same BookStack instance.

## LDAP Authentication (Active Directory / OpenLDAP)

For organizations with LDAP/AD, enable LDAP authentication in `.env`:

```bash
sudo nano /var/www/bookstack/.env
```

```env
# LDAP authentication settings
AUTH_METHOD=ldap

# LDAP server connection
LDAP_SERVER=ldap://ldap.example.com:389
LDAP_BASE_DN=dc=example,dc=com
LDAP_DN=cn=ldap-reader,dc=example,dc=com
LDAP_PASS=ldap-reader-password

# LDAP user search filter
LDAP_USER_FILTER=(&(uid=${user}))
# For Active Directory:
# LDAP_USER_FILTER=(&(sAMAccountName=${user}))

# Map LDAP attributes to BookStack fields
LDAP_VERSION=3
LDAP_USER_TO_GROUPS=true
LDAP_GROUP_ATTRIBUTE=memberOf
LDAP_REMOVE_FROM_GROUPS=true

# Email attribute
LDAP_EMAIL_ATTRIBUTE=mail
LDAP_DISPLAY_NAME_ATTRIBUTE=cn
```

```bash
# Restart PHP-FPM for changes to take effect
sudo systemctl restart php8.2-fpm
```

## Enabling Markdown Editor

BookStack defaults to the WYSIWYG editor. Enable Markdown per-user:

1. Click your profile (top right)
2. Edit Profile
3. Editor Preference: Markdown

Or set the default for all users in Settings > Application Settings > Default Editor.

## Setting Up Automatic Backups

```bash
# Create backup script
cat > /usr/local/bin/backup-bookstack.sh << 'SCRIPT'
#!/bin/bash
BACKUP_DIR="/backup/bookstack"
DATE=$(date +%Y%m%d)

mkdir -p "$BACKUP_DIR"

# Backup database
mysqldump -u bookstack -pstrong-db-password bookstack | \
  gzip > "$BACKUP_DIR/bookstack-db-${DATE}.sql.gz"

# Backup uploaded files (images and attachments)
tar -czf "$BACKUP_DIR/bookstack-uploads-${DATE}.tar.gz" \
  /var/www/bookstack/storage/uploads/

# Backup .env file (contains credentials - store securely)
cp /var/www/bookstack/.env "$BACKUP_DIR/bookstack-env-${DATE}"

# Remove backups older than 30 days
find "$BACKUP_DIR" -mtime +30 -delete

echo "BookStack backup completed: $(date)"
SCRIPT

chmod +x /usr/local/bin/backup-bookstack.sh

echo "0 3 * * * root /usr/local/bin/backup-bookstack.sh >> /var/log/bookstack-backup.log 2>&1" | \
  sudo tee /etc/cron.d/bookstack-backup
```

## Troubleshooting

**Blank page or 500 error:**
```bash
# Check Laravel application logs
sudo tail -f /var/www/bookstack/storage/logs/laravel.log

# Check PHP-FPM logs
sudo journalctl -u php8.2-fpm -f

# Check Nginx error logs
sudo tail -f /var/log/nginx/error.log
```

**Permission errors:**
```bash
sudo chown -R www-data:www-data /var/www/bookstack/storage /var/www/bookstack/bootstrap/cache
sudo chmod -R 775 /var/www/bookstack/storage /var/www/bookstack/bootstrap/cache
```

**File uploads failing:**
```bash
# Check upload size limits in PHP
php -i | grep upload_max_filesize
php -i | grep post_max_size

# Increase in /etc/php/8.2/fpm/php.ini:
# upload_max_filesize = 100M
# post_max_size = 100M
sudo systemctl restart php8.2-fpm
```

## Summary

BookStack on Ubuntu provides clean, structured documentation with an intuitive hierarchy and a WYSIWYG editor that doesn't require Markdown knowledge. The role-based permission system handles everything from fully open wikis to granular per-book access control. For engineering teams, the Markdown editor and Git-like URL structure for pages make it a comfortable alternative to heavyweight platforms like Confluence, without the licensing costs.
