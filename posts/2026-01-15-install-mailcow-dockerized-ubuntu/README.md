Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Description: Complete guide to installing Mailcow Dockerized mail server solution on Ubuntu.

---

Running your own mail server gives you full control over your email infrastructure, privacy, and data. Mailcow Dockerized is one of the most comprehensive and user-friendly self-hosted mail server solutions available. It bundles Postfix, Dovecot, SOGo, and many other components into a cohesive Docker-based stack. This guide walks you through the complete installation process on Ubuntu.

## 1. Understanding Mailcow

Mailcow Dockerized is an open-source mail server suite that packages everything you need to run a production-ready email system. Unlike traditional mail server setups that require configuring dozens of services manually, Mailcow provides a unified solution.

### What Mailcow Includes

- **Postfix**: SMTP server for sending and receiving mail
- **Dovecot**: IMAP/POP3 server for mail retrieval
- **SOGo**: Webmail and calendar/contacts (groupware)
- **Rspamd**: Advanced spam filtering with machine learning
- **ClamAV**: Antivirus scanning for attachments
- **Nginx**: Web server and reverse proxy
- **Redis**: Caching and session storage
- **MariaDB**: Database for storing configurations
- **Unbound**: DNS resolver for DNSSEC validation
- **ACME**: Automatic SSL/TLS certificate management via Let's Encrypt
- **Netfilter**: Fail2ban integration for brute-force protection

### Why Choose Mailcow?

- **All-in-one solution**: No need to configure individual components
- **Modern web UI**: Intuitive administration interface
- **Docker-based**: Easy deployment, updates, and isolation
- **Active development**: Regular security updates and new features
- **Strong community**: Extensive documentation and community support

## 2. Prerequisites

Before installing Mailcow, ensure your system meets the following requirements.

### Hardware Requirements

```bash
# Minimum recommended specifications
# - CPU: 2 cores (4+ recommended for production)
# - RAM: 4 GB minimum (6+ GB recommended)
# - Storage: 20 GB minimum (SSD recommended for better performance)
# - Network: Static public IP address with proper reverse DNS
```

### Operating System

This guide uses Ubuntu 22.04 LTS or Ubuntu 24.04 LTS. Ensure your system is updated:

```bash
# Update package lists and upgrade existing packages
sudo apt update && sudo apt upgrade -y

# Install essential utilities
sudo apt install -y curl git apt-transport-https ca-certificates gnupg lsb-release
```

### Docker Installation

Mailcow requires Docker and Docker Compose. Install them using the official Docker repository:

```bash
# Add Docker's official GPG key
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Add the Docker repository to APT sources
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Update package lists to include Docker packages
sudo apt update

# Install Docker Engine, CLI, containerd, and Docker Compose plugin
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Verify Docker installation - should show version info
docker --version

# Verify Docker Compose installation
docker compose version
```

Add your user to the docker group to run Docker commands without sudo:

```bash
# Add current user to docker group
sudo usermod -aG docker $USER

# Apply group changes (or log out and back in)
newgrp docker
```

### DNS Configuration

Proper DNS setup is critical for mail delivery. Configure the following DNS records for your domain (replace `mail.example.com` with your hostname and `203.0.113.10` with your server IP):

```
# A Record - Points your mail hostname to your server IP
mail.example.com.       IN  A       203.0.113.10

# MX Record - Tells other servers where to send mail for your domain
example.com.            IN  MX  10  mail.example.com.

# SPF Record - Specifies which servers can send mail for your domain
example.com.            IN  TXT     "v=spf1 mx a -all"

# DKIM Record - Will be generated by Mailcow (add after installation)
dkim._domainkey.example.com.  IN  TXT  "v=DKIM1; k=rsa; p=<public-key>"

# DMARC Record - Policy for handling failed authentication
_dmarc.example.com.     IN  TXT     "v=DMARC1; p=quarantine; rua=mailto:postmaster@example.com"

# PTR Record (Reverse DNS) - Must be configured by your hosting provider
# 10.113.0.203.in-addr.arpa.  IN  PTR  mail.example.com.

# Autodiscover records for automatic mail client configuration
autodiscover.example.com.    IN  CNAME   mail.example.com.
autoconfig.example.com.      IN  CNAME   mail.example.com.

# SRV records for automatic client configuration
_autodiscover._tcp.example.com.  IN  SRV  0 0 443 mail.example.com.
```

### Firewall Configuration

Open the required ports for mail services:

```bash
# Enable UFW firewall if not already enabled
sudo ufw enable

# Allow SSH (important: don't lock yourself out!)
sudo ufw allow 22/tcp

# HTTP and HTTPS for web interface and Let's Encrypt
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# SMTP - Mail submission and relay
sudo ufw allow 25/tcp       # SMTP (server-to-server)
sudo ufw allow 465/tcp      # SMTPS (submission over TLS)
sudo ufw allow 587/tcp      # Submission (STARTTLS)

# IMAP and POP3 - Mail retrieval
sudo ufw allow 143/tcp      # IMAP (STARTTLS)
sudo ufw allow 993/tcp      # IMAPS (IMAP over TLS)
sudo ufw allow 110/tcp      # POP3 (STARTTLS)
sudo ufw allow 995/tcp      # POP3S (POP3 over TLS)

# SOGo/CalDAV/CardDAV (optional, for calendar and contacts)
sudo ufw allow 4190/tcp     # ManageSieve

# Verify firewall rules
sudo ufw status verbose
```

## 3. Cloning Mailcow Repository

Mailcow should be installed in `/opt` for proper organization:

```bash
# Navigate to /opt directory (standard location for third-party software)
cd /opt

# Clone the Mailcow repository from GitHub
# This downloads all necessary Docker configurations and scripts
sudo git clone https://github.com/mailcow/mailcow-dockerized.git

# Change ownership to your user for easier management
sudo chown -R $USER:$USER mailcow-dockerized

# Enter the Mailcow directory
cd mailcow-dockerized
```

## 4. Configuration (mailcow.conf)

Run the interactive configuration script to generate the main configuration file:

```bash
# Run the generate_config.sh script
# This will prompt you for your mail server hostname
./generate_config.sh
```

The script will ask for your mail server's FQDN (Fully Qualified Domain Name). Enter your mail hostname:

```
Mail server hostname (FQDN) - this is NOT your mail domain, but your mail servers hostname: mail.example.com
```

After the script completes, review and customize the generated configuration:

```bash
# Open mailcow.conf for editing
nano mailcow.conf
```

Key configuration options to review:

```bash
# mailcow.conf - Main configuration file

# Your mail server's hostname (FQDN)
# This MUST resolve to your server's IP address
MAILCOW_HOSTNAME=mail.example.com

# Branch to use (master for stable, nightly for latest features)
MAILCOW_BRANCH=master

# Timezone - Set to your local timezone
# Use 'timedatectl list-timezones' to see available options
TZ=America/New_York

# HTTP and HTTPS bind ports
# Change these if you have another web server on ports 80/443
HTTP_PORT=80
HTTPS_PORT=443

# HTTP bind IP address (0.0.0.0 for all interfaces)
HTTP_BIND=0.0.0.0
HTTPS_BIND=0.0.0.0

# SMTP ports
SMTP_PORT=25
SMTPS_PORT=465
SUBMISSION_PORT=587

# IMAP/POP3 ports
IMAP_PORT=143
IMAPS_PORT=993
POP_PORT=110
POPS_PORT=995

# Sieve port (for email filtering rules)
SIEVE_PORT=4190

# Skip Let's Encrypt certificate generation
# Set to 'y' if you want to use your own certificates
SKIP_LETS_ENCRYPT=n

# Skip IP check (useful if behind NAT or load balancer)
SKIP_IP_CHECK=n

# Enable ClamAV antivirus scanning
# Set to 'n' to disable (saves ~1GB RAM)
SKIP_CLAMD=n

# Enable SOGo groupware (webmail, calendar, contacts)
SKIP_SOGO=n

# Enable Solr full-text search (requires additional RAM)
SKIP_SOLR=y

# Compose project name (don't change unless you know what you're doing)
COMPOSE_PROJECT_NAME=mailcowdockerized

# Additional SAN (Subject Alternative Names) for SSL certificates
# Add additional hostnames separated by commas
ADDITIONAL_SAN=autodiscover.example.com,autoconfig.example.com

# API key for admin access (auto-generated, keep secret)
API_KEY=your-generated-api-key

# Enable API access
API_ALLOW_FROM=127.0.0.1,::1

# Docker Compose compatibility
DOCKER_COMPOSE_VERSION=native
```

### Memory Optimization for Smaller Servers

If you're running on a server with limited RAM, consider these optimizations:

```bash
# Edit mailcow.conf to disable resource-intensive components

# Disable ClamAV (saves ~1GB RAM, but no antivirus scanning)
SKIP_CLAMD=y

# Disable Solr (saves ~512MB RAM, but no full-text search)
SKIP_SOLR=y

# Optionally disable SOGo if you don't need webmail
# SKIP_SOGO=y
```

## 5. Starting Mailcow

Pull the Docker images and start all containers:

```bash
# Navigate to the Mailcow directory
cd /opt/mailcow-dockerized

# Pull all required Docker images (this may take several minutes)
# The images total approximately 2-3 GB
docker compose pull

# Start all Mailcow containers in detached mode
docker compose up -d

# Verify all containers are running
# You should see about 15+ containers with "Up" status
docker compose ps
```

Monitor the startup process:

```bash
# View logs from all containers (Ctrl+C to exit)
docker compose logs -f

# Or view logs from specific containers
docker compose logs -f postfix-mailcow     # SMTP server logs
docker compose logs -f dovecot-mailcow     # IMAP server logs
docker compose logs -f nginx-mailcow       # Web server logs
docker compose logs -f rspamd-mailcow      # Spam filter logs
```

## 8. SSL/TLS Certificates

Mailcow uses Let's Encrypt for automatic certificate management by default.

### Automatic Certificates (Let's Encrypt)

```bash
# Certificates are automatically obtained when:
# 1. Port 80 is accessible from the internet
# 2. DNS records point to your server
# 3. SKIP_LETS_ENCRYPT=n in mailcow.conf

# Force certificate renewal
cd /opt/mailcow-dockerized
docker compose exec acme-mailcow /etc/cron.daily/acme
```

### Check Certificate Status

```bash
# View current certificates
docker compose exec acme-mailcow /etc/cron.daily/acme --status

# View certificate expiration dates
docker compose exec nginx-mailcow openssl x509 -in /etc/ssl/mail/cert.pem -noout -dates
```

### Using Custom Certificates

If you prefer to use your own certificates:

```bash
# 1. Set SKIP_LETS_ENCRYPT=y in mailcow.conf

# 2. Place your certificates in the following location:
# /opt/mailcow-dockerized/data/assets/ssl/

# Required files:
# - cert.pem (certificate)
# - key.pem (private key)
# - chain.pem (intermediate certificates, if any)

# 3. Set correct permissions
sudo chmod 600 /opt/mailcow-dockerized/data/assets/ssl/key.pem
sudo chmod 644 /opt/mailcow-dockerized/data/assets/ssl/cert.pem

# 4. Restart Mailcow to apply
docker compose restart
```

### Additional SANs for Subdomains

To include additional hostnames in your certificate:

```bash
# Edit mailcow.conf
nano /opt/mailcow-dockerized/mailcow.conf

# Add additional Subject Alternative Names
ADDITIONAL_SAN=autodiscover.example.com,autoconfig.example.com,webmail.example.com

# Restart and renew certificates
docker compose down
docker compose up -d
```

## 10. Backup and Restore

Regular backups are essential for any mail server.

### Backup Script

Mailcow includes a built-in backup helper:

```bash
# Navigate to Mailcow directory
cd /opt/mailcow-dockerized

# Create a backup directory
sudo mkdir -p /backup/mailcow

# Run the backup script
# This backs up: vmail (mailboxes), crypt (encryption keys),
# redis, rspamd, postfix, mysql (database)
./helper-scripts/backup_and_restore.sh backup all

# The backup will be stored in the default location
# Or specify a custom path:
BACKUP_LOCATION=/backup/mailcow ./helper-scripts/backup_and_restore.sh backup all
```

### Automated Backups

Create a cron job for automated backups:

```bash
# Create a backup script
sudo nano /opt/mailcow-dockerized/backup.sh
```

Add the following content:

```bash
#!/bin/bash
# Mailcow automated backup script

# Set backup location (change as needed)
export BACKUP_LOCATION=/backup/mailcow

# Set retention (number of backups to keep)
RETENTION_DAYS=7

# Navigate to Mailcow directory
cd /opt/mailcow-dockerized

# Run backup
./helper-scripts/backup_and_restore.sh backup all

# Remove backups older than retention period
find $BACKUP_LOCATION -type d -mtime +$RETENTION_DAYS -exec rm -rf {} \; 2>/dev/null

# Log completion
echo "Backup completed at $(date)" >> /var/log/mailcow-backup.log
```

Make it executable and add to cron:

```bash
# Make script executable
sudo chmod +x /opt/mailcow-dockerized/backup.sh

# Add to crontab (runs daily at 3 AM)
sudo crontab -e

# Add this line:
0 3 * * * /opt/mailcow-dockerized/backup.sh
```

### Restore from Backup

```bash
# Navigate to Mailcow directory
cd /opt/mailcow-dockerized

# List available backups
BACKUP_LOCATION=/backup/mailcow ./helper-scripts/backup_and_restore.sh list

# Restore all components from a specific backup
BACKUP_LOCATION=/backup/mailcow ./helper-scripts/backup_and_restore.sh restore all

# Or restore specific components:
# Restore only mailbox data
BACKUP_LOCATION=/backup/mailcow ./helper-scripts/backup_and_restore.sh restore vmail

# Restore only database
BACKUP_LOCATION=/backup/mailcow ./helper-scripts/backup_and_restore.sh restore mysql
```

### Manual Database Backup

For additional database security:

```bash
# Backup MySQL database manually
docker compose exec mysql-mailcow mysqldump --all-databases -u root -p > /backup/mailcow-db-$(date +%Y%m%d).sql

# Restore MySQL database
docker compose exec -T mysql-mailcow mysql -u root -p < /backup/mailcow-db-20240115.sql
```

## 11. Updating Mailcow

Keep your Mailcow installation secure with regular updates.

### Update Process

```bash
# Navigate to Mailcow directory
cd /opt/mailcow-dockerized

# ALWAYS backup before updating
./helper-scripts/backup_and_restore.sh backup all

# Fetch the latest changes from the repository
git fetch origin

# Check for available updates
./update.sh --check

# Run the update script
# This will pull new images and restart containers
./update.sh

# The script will show changes and ask for confirmation
# Review the changes before proceeding
```

### Update Script Options

```bash
# Check for updates without applying
./update.sh --check

# Force update (skip confirmations - use with caution)
./update.sh --force

# Update to a specific version
git fetch --tags
git checkout <version-tag>
docker compose pull
docker compose up -d
```

## 12. Troubleshooting

Common issues and their solutions.

### Container Issues

```bash
# Check container status
docker compose ps

# Restart a specific container
docker compose restart postfix-mailcow

# Restart all containers
docker compose restart

# View container logs
docker compose logs -f <container-name>

# Common container names:
# postfix-mailcow, dovecot-mailcow, nginx-mailcow
# rspamd-mailcow, clamd-mailcow, mysql-mailcow
# sogo-mailcow, unbound-mailcow, redis-mailcow
```

### Mail Delivery Problems

```bash
# Check Postfix queue
docker compose exec postfix-mailcow mailq

# Flush the mail queue (resend all queued mail)
docker compose exec postfix-mailcow postqueue -f

# Delete all queued mail (use with caution!)
docker compose exec postfix-mailcow postsuper -d ALL

# Check Postfix logs for errors
docker compose logs postfix-mailcow | grep -i error

# Test SMTP connection
telnet mail.example.com 25
# Or using OpenSSL for TLS
openssl s_client -connect mail.example.com:465
```

### Authentication Issues

```bash
# Check Dovecot authentication logs
docker compose logs dovecot-mailcow | grep -i auth

# Verify user exists in database
docker compose exec mysql-mailcow mysql -u mailcow -p mailcow -e "SELECT username FROM mailbox;"

# Reset admin password
cd /opt/mailcow-dockerized
./helper-scripts/admin_reset_password.sh
```

### DNS Issues

```bash
# Test DNS resolution inside containers
docker compose exec unbound-mailcow dig example.com

# Check if DNS records are correct
dig +short MX example.com
dig +short A mail.example.com
dig +short TXT example.com
dig +short TXT _dmarc.example.com

# Test SPF record
docker compose exec postfix-mailcow postconf -n | grep smtpd_sender_restrictions
```

### SSL Certificate Issues

```bash
# Check certificate status
docker compose exec acme-mailcow /etc/cron.daily/acme --status

# Force certificate renewal
docker compose exec acme-mailcow /etc/cron.daily/acme --force

# View certificate details
docker compose exec nginx-mailcow openssl x509 -in /etc/ssl/mail/cert.pem -text -noout

# Check for Let's Encrypt errors
docker compose logs acme-mailcow
```

### Spam Filter Issues

```bash
# Check Rspamd status
docker compose exec rspamd-mailcow rspamadm configtest

# View Rspamd statistics
curl http://localhost:11334/stat

# Access Rspamd web interface
# https://mail.example.com/rspamd/

# Train spam filter
# Ham (good email): Move to Inbox
# Spam: Move to Junk folder
# Rspamd learns automatically from these actions
```

### Performance Issues

```bash
# Check container resource usage
docker stats

# Check disk space
df -h

# Check mail storage usage
du -sh /opt/mailcow-dockerized/data/vmail/

# Reduce memory usage by disabling ClamAV
# Edit mailcow.conf: SKIP_CLAMD=y
# Then: docker compose down && docker compose up -d
```
