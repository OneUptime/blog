## Introduction

Bash shell completion for cilium-bugtool enables tab completion of subcommands, flags, and arguments directly in your terminal. This reduces the need to consult help text and speeds up diagnostic workflows when collecting Cilium debugging data.




The bash completion script generated by cilium-bugtool contains the full command tree encoded as shell functions and case statements. By parsing this script, you can extract the complete list of subcommands, flags, and their relationships.

This guide covers parsing techniques for extracting structured data from bash completion output.

## Prerequisites

- Bash shell (v4.0+)
- `bash-completion` package installed
- `cilium-bugtool` binary available locally or in a Cilium pod
- `kubectl` access to a Cilium cluster (if binary is not local)

## Capturing the Completion Script




\`\`\`bash
# Generate and save the completion script
cilium-bugtool completion bash > /tmp/bugtool-bash-completion.sh

# Check file size
wc -l /tmp/bugtool-bash-completion.sh
\`\`\`

### Extracting Commands

\`\`\`bash
# Extract registered command names
grep -oP 'commands=\(\s*"\K[^"]+' /tmp/bugtool-bash-completion.sh | sort -u

# Extract from case statements
grep -oP "^\s+'[a-z][-a-z]*'\)" /tmp/bugtool-bash-completion.sh |   tr -d " ')" | sort -u
\`\`\`

### Extracting Flags

\`\`\`bash
# Extract all long flags
grep -oP '\-\-[a-z][-a-z0-9]*' /tmp/bugtool-bash-completion.sh | sort -u

# Extract flags with their associated commands
awk '/^__.*_flag_/ {cmd=$0} /--[a-z]/ {print cmd, $0}'   /tmp/bugtool-bash-completion.sh | head -20
\`\`\`

### Python-Based Parser

\`\`\`python
#!/usr/bin/env python3
"""Parse cilium-bugtool bash completion for command and flag extraction."""

import re
import json
import sys

def parse_bash_completion(filepath):
    with open(filepath) as f:
        content = f.read()

    # Extract all unique flags
    flags = sorted(set(re.findall(r'--[a-z][-a-z0-9]+', content)))

    # Extract commands from COMPREPLY and case blocks
    commands = sorted(set(re.findall(r'"([a-z][-a-z]*)"', content)))

    # Extract function names that handle specific commands
    handlers = sorted(set(re.findall(r'__cilium[-_]bugtool[-_](\w+)', content)))

    return {
        'flags': flags,
        'commands': commands,
        'handlers': handlers,
        'total_lines': content.count('\n')
    }

if __name__ == '__main__':
    path = sys.argv[1] if len(sys.argv) > 1 else '/tmp/bugtool-bash-completion.sh'
    result = parse_bash_completion(path)
    print(json.dumps(result, indent=2))
\`\`\`

### Generating Documentation

\`\`\`bash
# Convert parsed output to markdown
python3 parse_bash_completion.py /tmp/bugtool-bash-completion.sh |   jq -r '.flags[]' | while read -r flag; do
    echo "- \\\`\$flag\\\`"
  done > /tmp/bugtool-flags.md

echo "Generated flag documentation at /tmp/bugtool-flags.md"

## Troubleshooting

- **Regex mismatches**: Bash completion format varies by Cobra version. Inspect the raw file first.
- **Empty command list**: Try alternative patterns; the structure depends on the CLI framework version.
- **Python script errors on encoding**: Open files with \`errors='ignore'\` for robustness.
- **Missing flags in parsed output**: Some flags may be defined dynamically and not appear in the static script.
