## Introduction

Open source projects thrive when community members contribute to improving processes and resources. Extending the Cilium project roadmap helps the entire Cilium community by making information more accessible and processes more efficient.

Contributing to roadmap does not require deep technical expertise -- clear communication, attention to detail, and understanding of community needs are equally valuable.

This guide covers how to contribute to and extend the Cilium project roadmap.

## Prerequisites

- Familiarity with the Cilium project and its ecosystem
- A GitHub account for participating in project discussions
- Understanding of Cilium architecture and features

## Contributing to the Roadmap

### Proposing Features

1. Check if a similar proposal exists in GitHub issues
2. Start a discussion in Slack or community meetings
3. Write a design proposal if the idea gains support
4. Submit through the Cilium Enhancement Proposal process

### Providing Feature Feedback

Help shape the roadmap through:
- Commenting on GitHub issues with use cases
- Voting on feature requests (thumbs up reactions)
- Sharing production experience that validates need
- Offering to test or review implementations

### Influencing Priorities

Demonstrate feature importance by:
- Documenting real-world use cases
- Quantifying the impact on your deployment
- Offering engineering resources for development
- Building proof-of-concept implementations

### Community Roadmap Discussions

Participate in roadmap planning:
- Attend community meetings during planning cycles
- Join SIG meetings for area-specific roadmap items
- Respond to roadmap surveys when published
- Share organizational deployment plans (anonymized if needed)

## Verification

Verify roadmap information is accessible and up to date.

## Troubleshooting

- **Cannot find meeting links**: Check the Cilium community calendar and #community Slack channel.
- **Slack workspace access**: Request an invite through the Cilium website.
- **GitHub permissions**: Ensure your account has the necessary access for the repositories you need.
- **Timezone confusion**: All official times are in UTC. Use a timezone converter for your local time.

## Conclusion

The project roadmap provides opportunities to understanding project direction. Active participation strengthens both your own Cilium practice and the broader community.
