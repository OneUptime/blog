## Introduction

The `cilium-bugtool` command includes a `completion` subcommand that generates shell completion scripts. These completions enable tab-completion of cilium-bugtool subcommands, flags, and arguments in your terminal, making diagnostic workflows faster and less error-prone.




The completion scripts generated by cilium-bugtool contain structured information about the CLI's command tree, flags, and their descriptions. Parsing this output enables automated documentation generation, CLI coverage validation, and custom tooling.

This guide covers techniques for extracting structured data from cilium-bugtool completion output.

## Prerequisites

- `cilium-bugtool` binary available locally or in a Cilium pod
- Shell environment (Bash 4+, Zsh 5+, Fish 3+, or PowerShell 5+)
- `kubectl` access to a Cilium cluster (if binary is not local)

## Capturing the Completion Output




\`\`\`bash
# Capture completion output for analysis
cilium-bugtool completion bash > /tmp/bugtool-completion.bash
cilium-bugtool completion zsh > /tmp/bugtool-completion.zsh
\`\`\`

### Extracting Subcommands

\`\`\`bash
# From bash completion, extract registered commands
grep -oP "commands=\(\K[^)]*" /tmp/bugtool-completion.bash |   tr ' ' '\n' | sed 's/"//g' | sort -u

# From zsh completion, extract command descriptions
grep -oP "'[a-z][-a-z]*\[.*?\]" /tmp/bugtool-completion.zsh |   sed "s/'//g" | sort -u
\`\`\`

### Extracting Flags

\`\`\`bash
# Extract all flags from bash completion
grep -oP '\-\-[a-z][-a-z0-9]*' /tmp/bugtool-completion.bash | sort -u

# Extract flags with descriptions from zsh completion
grep -oP "'--[a-z][-a-z0-9]*\[.*?\]" /tmp/bugtool-completion.zsh |   sed "s/'//g;s/\[/: /;s/\]//" | sort -u
\`\`\`

### Python Parser

\`\`\`python
#!/usr/bin/env python3
"""Parse cilium-bugtool completion output."""
import re
import json
import sys

def parse_bash_completion(filepath):
    with open(filepath) as f:
        content = f.read()
    flags = sorted(set(re.findall(r'--[a-z][-a-z0-9]*', content)))
    commands_raw = re.findall(r'commands=\(([^)]*)\)', content)
    commands = []
    for block in commands_raw:
        commands.extend(re.findall(r'"([^"]+)"', block))
    return {'flags': flags, 'commands': sorted(set(commands))}

if __name__ == '__main__':
    filepath = sys.argv[1] if len(sys.argv) > 1 else '/tmp/bugtool-completion.bash'
    result = parse_bash_completion(filepath)
    print(json.dumps(result, indent=2))
\`\`\`

## Verification

## Troubleshooting

- **Regex finds no matches**: The completion output format varies. Check the raw file and adjust patterns.
- **Duplicate entries**: Add deduplication with \`sort -u\` or Python \`set()\`.
- **Missing commands in parsed output**: Some commands may be registered dynamically. Compare with \`cilium-bugtool --help\`.
- **JSON output is invalid**: Escape special characters in command descriptions before JSON serialization.
